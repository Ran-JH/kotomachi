# Kotomachi 手动验收清单（H1 / I1 / J1）

适用范围：表达提示（H1）、查词解释（I1）、回顾卡片（J1）联动验收。  
执行方式：按顺序勾选；如失败，记录输入句子、NPC、时间和截图。

## 0. 预检查

- [ ] 进入任一 NPC 聊天页，可正常收发消息。
- [ ] 当前会话至少有 2 条用户消息（用于回顾卡片生成）。
- [ ] 准备 2 组测试输入：
  - [ ] 含 ASR 断裂/混合语输入（例如：`にな-になった`、英文残句）。
  - [ ] 常规日语输入（例如：`明日レポートを書かないといけない。`）。

## 1. 表达提示（H1）

### 1.1 三档差异

- [ ] 点击用户消息的 `💡 提案` 打开表达提示。
- [ ] `亲近自然 / 普通自然 / 更礼貌` 三档不是同一句，仅尾部 `です/ます` 变化不算通过。
- [ ] 三档均与原句意图一致，不出现牛头不对马嘴内容。

### 1.2 ASR 断裂与混合语修复

- [ ] 输入含断裂文本（如：`もう言わないで、私は今すぐって寝たいにな-になった。`）后打开表达提示。
- [ ] 推荐句不保留 `にな-になった` 这类断裂片段。
- [ ] 英文/中英混合输入会先还原意图，再给自然日语，不机械直译残句。

### 1.3 听一下播放

- [ ] 三档 `听一下` 均可播放，且分别播放对应档位推荐句。
- [ ] 用户录音回放按钮可用，且不遮挡三档内容。

## 2. 查词解释（I1）

### 2.1 结构完整

- [ ] 在 `夜勤の誘惑の方が強いよね。` 中划选 `夜勤`：
  - [ ] 有读音（`やきん`）。
  - [ ] 简义是“夜班/夜间工作”而不是整句翻译。
  - [ ] 有“这句话里的意思”。
- [ ] 同句划选 `誘惑`：
  - [ ] 简义准确（诱惑/吸引力）。
  - [ ] 句中意思与当前句子一致。
- [ ] 在 `もう言わないで、私は今すぐ寝たい。` 中划选 `寝たい`：
  - [ ] 简义准确（想睡觉）。
  - [ ] 语感/说明提到愿望表达（`たい`）。

### 2.2 展开解释与失败兜底

- [ ] 有实质详情时才显示 `展开解释 / 收起解释`。
- [ ] 模拟详情失败时：
  - [ ] 基础解释（读音/简义/句中意思）仍保留。
  - [ ] 错误提示温和，不出现上下文断裂。
  - [ ] 不出现无意义展开按钮。

## 3. 回顾卡片（J1）

### 3.1 标题质量

- [ ] 生成回顾卡片后，标题是话题总结，不是英文残句或原句截断。
- [ ] 以下坏例不得出现：`Sorry i feel f...の話`。
- [ ] 类似好例可通过：`試験とレポートの話`、`夜勤と眠気の話`。

### 3.2 今日の話 / 今天聊了什么

- [ ] 内容真实概括本轮对话，不空泛、不编造没聊过的主题。

### 3.3 次はこう言える / 下次可以这样说

- [ ] 每条包含 `original`、`suggestion`、`note`、`source`。
- [ ] 有非日语输入时，优先使用该片段生成表达升级。
- [ ] 无非日语时，优先利用用户打开/播放过的表达提示信号。
- [ ] 建议句自然可复用，不机械翻译英文残句。

### 3.4 今日のことば / 今日词语

- [ ] 有 recentLookups 时，优先出现查过的词（如 `夜勤`、`誘惑`）。
- [ ] `source` 正确标注为 `looked_up` 或 `conversation`。
- [ ] 词条是词或短语，不是完整句子。
- [ ] 尽量有 `reading`、必须有 `meaning`，`example` 合理可读。

### 3.5 次に話してみること / 下次可以聊

- [ ] 话题具体、可继续聊，不是“继续聊这个话题”。
- [ ] 示例通过方向：夜勤后休息方式、报告选题方式、考试前确认方式。

### 3.6 重复生成保护

- [ ] 同一批消息重复点生成，不新增重复卡片。
- [ ] 行为为“打开已有卡片”或“提示已生成”。

## 4. 交叉验证（学习信号 -> 回顾卡片）

- [ ] 先做一次查词（如 `夜勤`/`誘惑`），再生成回顾卡片，确认词语区优先引用查词结果。
- [ ] 先打开并播放表达提示，再生成回顾卡片，确认表达升级可体现相关学习信号。
- [ ] 对话中含英文残句时，确认表达升级优先处理该片段，而非随机挑句。

## 5. 其他核心链路（不回归）

- [ ] TTS：NPC 语音可正常播放。
- [ ] STT：录音转文字可用，空录音不破坏聊天主流程。
- [ ] 语言切换：仅 UI 文案切换，NPC 回复/用户输入/学习内容不被翻译。
- [ ] 时间分隔条：消息区使用稀疏时间分隔，不回到“每条都显示时间”。
- [ ] welcome：重复进入同一会话不出现无意义重复 welcome。

# 表达提示 / 查词解释 / 回顾卡片质量规格

## 用途

这份文档用于约束 Kotomachi / 言街 中三个学习辅助层的内容质量：

- `💡 提案` / 表达提示
- 划词查词解释
- `ふりかえり` / 回顾卡片

它不是新功能需求，而是后续修 prompt、fallback、解析逻辑和 UI 展示时的质量边界。目标是避免为了“低压力”而输出低信息量、跑题或错误的学习内容。

## 总原则

低压力不等于低信息量。

Kotomachi 的主聊天层可以轻松、短、像 LINE 聊天；但学习辅助层既然是用户主动打开的，就必须提供真实可复用的信息。温和语气不能变成空泛鼓励，简短不能变成无效内容。

学习内容必须 grounded，至少基于以下一种真实信号：

- 用户当前原句或语音识别文本。
- 用户划选的词和该词所在完整句子。
- 用户最近真实对话。
- 用户主动查过的词。
- 用户打开或播放过的表达提示。
- 用户输入中的中文、英文或混合语言片段。

不能为了填满结构而捏造用户没说过、没查过、没关注过的学习点。

## 表达提示质量标准

表达提示必须先理解用户意图，再给自然日语。

如果用户说的是“我想点咖啡”，建议必须围绕点咖啡；如果用户问“扱う是什么意思”，建议必须围绕询问词义；如果用户输入中有 ASR 断裂，例如 `にな-になった`，应先还原意图，不能机械保留断裂文本。

三档建议必须真实区分语气和场景：

- 亲近自然 / カジュアル：朋友、熟人、轻松聊天；可以短，可以更口语。
- 普通自然 / ふつう：店员、同事、邻居、日常社交；礼貌但不僵硬。
- 更礼貌 / フォーマル：上级、客户、面试、书面；结构更完整，敬语更稳。

三档不能只是同一句话加减 `です/ます`，也不能完全相同。每档的推荐句必须能直接播放、模仿和复用。

注释必须解释“为什么这个场合这样说”和“原句哪里可以调整”。不要只写“更自然”“更礼貌”“适合正式场合”这种空话；也不要乱解释不存在的语法问题。

## 查词解释质量标准

查词解释必须区分三层：

- 词义：被划选词或短语本身的简短中文意思。
- 句中意思：这个词在当前完整句子里让整句话是什么意思。
- 语感 / 使用场景：这个表达在口语、熟人聊天、正式场合等语境里的感觉。

不能把整句话直接当成词义。比如用户划选 `間違え`，词义应是“弄错 / 错误”，不能把整句翻译塞进简义区。

日期、数字等有特殊读法的表达，必须给出正确读音，不能按字面拆解。例如 `6/20` 应读作 `ろくがつはつか`，不能读作 `ろくがつにじゅうにち`。

如果详情解释失败，基础解释也必须保留；不能出现“简义能显示，但展开区只显示解释失败”的断裂体验。失败提示可以温和出现，但不能替代已有的词义和句中意思。

“展开解释 / 收起解释”只应该在确实有额外语感说明时出现。短解释、空详情、错误文案不应该触发展开按钮。

## 回顾卡片质量标准

回顾卡片必须是一次对话后的轻量复习卡，不是装饰性总结。

标题必须总结话题，不能直接截取英文残句、ASR 断裂文本或无意义片段。标题可以短，但要让用户一眼知道这张卡大概对应哪次对话。

“今日词语”只能放词或短语，不能放整句。可以是用户查过的词，也可以是对话中出现的实用短语；但不能把一整句用户输入塞进词语区。

“下次可以这样说”应优先来自用户真实表达缺口，例如中文/英文片段、ASR 断裂后可还原的意图、用户主动打开过的表达提示。不能泛泛写“继续用日语聊聊吧”。

“下次可以聊”必须具体、可继续聊。它应该能自然引出下一轮对话，例如围绕刚才的话题延伸一个轻问题；不能只写“下次继续聊这个话题”或“继续练习日语”。

## 不合格模式

以下输出视为 bad case：

- 表达提示三档几乎相同。
- 表达提示保留 ASR 断裂文本，如 `にな-になった`。
- 表达提示误读用户意图，给出牛头不对马嘴的三句。
- 表达提示注释空泛、乱解释或解释了用户没问的问题。
- 查词解释把整句话当成词义。
- 查词简义不准确，或缺少句中意思。
- 查词详情失败时覆盖基础解释。
- 回顾卡片标题直接截取英文残句。
- 回顾卡片把整句当成“今日词语”。
- 回顾卡片“下次可以聊”泛泛而谈。

## 后续分包计划

### Pack H1：只修表达提示

范围只包括 `💡 提案` 的 prompt、解析、fallback 和必要 UI 呈现。目标是让三档建议理解意图、区分场景、清理 ASR 断裂，并保持“听一下”播放对应句子。

不要同时修查词解释和回顾卡片。

### Pack I1：只修查词解释

范围只包括划词查词的 prompt、解析、fallback 和 popover 展示。目标是稳定区分词义、句中意思、语感说明，并处理详情失败不破坏基础解释。

不要同时修表达提示和回顾卡片。

### Pack J1：只修回顾卡片质量

范围只包括回顾卡片生成质量、结构化输出、fallback 和相关 evidence 选择。目标是修正标题、词语区、表达升级和下次话题的质量。

不要同时修表达提示和查词解释。
